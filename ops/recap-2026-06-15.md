## 🔴 THE 事故 + 治本(最重要的教训)
**经过**:连续 3×sync + 回填 6695 README + 多次 deploy,把免费 Nano 实例的**磁盘 IO 突发额度烧光** → 限流,降级 ~3.5h,连 `SELECT id LIMIT 1` 都要 6–12s,deploy 反复 57014。CPU 图证实是 **IOwait 打满**(等磁盘),不是计算瓶颈。

**治本**:
1. Supabase 升档 Nano→Micro(IO 基线 + 内存翻倍)—— 升完 deploy 一次过、查询回毫秒级
2. build 链路加固:catalog 拉取去 `readme_content`、页 1000→300、author-pages 深 OFFSET→keyset、scan 改分批 commit

**纪律(已写进 memory)**:`dont-overload-supabase` —— 串行所有 DB 重活,绝不重叠 sync/deploy/backfill,DB 慢时绝不硬上。
