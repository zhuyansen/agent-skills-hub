# AgentSkillsHub

The open-source directory for Claude Skills, MCP Servers, Codex Skills, and AI agent tools — indexed from GitHub, quality-scored, refreshed every 8 hours. This file pins the project's vocabulary so every Claude session uses the same words for the same things.

## Language

### Catalog & content

**Skill**:
A unit indexed by the Hub: a GitHub repo containing a `SKILL.md` (Anthropic spec), a Claude/Codex skill folder, an MCP server, an agent-tool, or any other Claude-Code-installable artifact.
_Avoid_: "tool", "plugin", "package", "repo" — they all mean different things below.

**MCP Server**:
A Skill whose `category = "mcp-server"`. Implements the Model Context Protocol; runs as a separate process. Currently 37% of the Hub.
_Avoid_: "MCP plugin", "MCP integration".

**Claude Skill**:
A Skill whose `category = "claude-skill"`. Loaded via `~/.claude/skills/{name}/SKILL.md`. Currently 16% of the Hub.
_Avoid_: "Claude plugin", "Anthropic skill".

**Category**:
One of 7 official taxonomic labels assigned by the sync pipeline: `mcp-server`, `claude-skill`, `codex-skill`, `agent-tool`, `ai-coding-assistant`, `prompt-library`, `uncategorized`.
_Avoid_: "type", "tag", "kind".

**Repo**:
The underlying GitHub repository. The Hub never hosts code — it indexes Repos.
_Avoid_: "project", "package".

### Quality & ranking

**Quality Score**:
Composite 0–100 from 6 dimensions (completeness, clarity, specificity, examples, README structure, agent readiness). Computed by `quality_analyzer.py`. Tiers: S ≥ 80, A 65–79, B 50–64, C 35–49, D < 35.
_Avoid_: "quality", "rating", "grade" — those are ambiguous.

**Score** (a.k.a. composite score):
0–100 weighted from 10 signals: stars, recency, forks, commits, issue resolution, momentum, author followers, size bonus, language coverage, plus quality. Used for default ranking.
_Avoid_: "rank" (rank is the position, score is the value).

**Star Velocity**:
Computed as `(stars - prev_stars)` over the last sync window. There is no `star_velocity` column — always derive it.
_Avoid_: "trending score".

### People & program

**Verified Creator**:
A Skill author who has earned (not bought) a verified badge by hitting `verified_score ≥ 65`. Formula: `0.40 × iteration + 0.40 × quality + 0.20 × community`. Detailed in Blue Book Ch10.
_Avoid_: "verified user", "premium author", "founding member".

**Master**:
A row in the `skill_masters` table — typically a Verified Creator with bio + X handle + curated repos. Surfaced on the Masters tab.
_Avoid_: "expert", "VIP", "creator-plus".

### Distribution & ops

**Distribution**:
The fourth edge of the商业化三角 (Blue Book Ch9). Hub's primary value prop: connecting Skill authors to users. Not "marketing" — Distribution = the channel itself.
_Avoid_: "promotion", "marketing".

**IndexNow**:
The Bing/Yandex protocol Hub uses to push fresh URLs after every deploy. Stable key in `public/{key}.txt`, never regenerate.
_Avoid_: "submit to search engines" (vague).

**Newsletter** vs **日报** vs **周报**:
- **Newsletter** = the verified-subscriber email sent by `newsletter_runner.py`, Mondays only
- **日报** = daily Top 10 picks generated by Claude+memory archive, posted to X
- **周报** = same data source as Newsletter but framed for X
_Avoid_: using these interchangeably.

## Relationships

- A **Repo** becomes a **Skill** once indexed by the sync pipeline
- A **Skill** has exactly one **Category** + a **Quality Score** + a **Score**
- A **Skill** belongs to one **Author**; an **Author** can become a **Verified Creator**, then become a **Master**
- The Hub's product is **Distribution** for Skills (not the Skills themselves)
- **Newsletter** and **日报** consume different DB views but share subscribers' attention budget

## Example dialogue

> **Dev:** "Should we boost a Verified Creator's Score in ranking?"
> **Jason:** "No — Verified gives a badge, not score weighting. The Score is the signal users trust; if we tilt it for Verifieds, the badge stops meaning anything."

> **Dev:** "User asked why the daily report missed a hot Skill — was it the Quality Score?"
> **Jason:** "Probably Star Velocity — the daily picks lean on `(stars - prev_stars)`, not Score. A great old Skill won't surface unless it's gaining stars now."

## Flagged ambiguities

- "trending" was used for both **weekly_trending_snapshots** (curated) and **star velocity** (raw delta) — resolved: "Trending Page" = the snapshot, "trending velocity" = the raw signal
- "creator" vs "author" — resolved: **Author** = whoever owns the GitHub repo (factual); **Creator** = an author who is part of the Verified Creator program (programmatic)
- "score" alone is ambiguous — always say **Score** (composite) or **Quality Score** (review)
